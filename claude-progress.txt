Key findings for the next agent:
- The design is intended as a minimal refactor, not a rewrite: reuse `AgentLoopManager + AsyncProduceStrategy + ReplayBuffer`.
- The shared strategy interface matters now, not just the async strategy: `ProduceStrategy` base class and `SyncProduceStrategy` must stay aligned with the manager refactor so colocate and disaggregated paths do not diverge.
- The core runtime split is:
  - producer-only path: `_produce_batch_to_buffer(...)`
  - explicit stop/reap path: `cleanup_pending_tasks(...)`
  - consumer-only path: `_get_batch_from_buffer(...)`
- The most important semantic change is that rollout freshness must track `model_rollout_step`, not the producer loop's local iteration count.
- `RLColocateTrainer` is now an explicit compatibility target, not just something indirectly covered by shared tests.
- `EXPIRED_BATCH` is intentionally aggressive: once rollout weights are stale enough, skip training and prioritize weight sync immediately.
- Checkpoints must be taken only after producer cleanup and before `reset()`, with the saved rollout step overridden to the post-sync target version.
- Resume should restore the manager into a paused `UPDATE_ABORT` state, re-sync rollout weights from train, then call `reset()` and start a fresh producer task.
- Colocate regression coverage should focus first on:
  - `examples/v1/config/rl_grpo_gsm8k_judge.py`
  - `examples/v1/config/rl_multi_task_gsm8k_dapo_math.py`
  - launch wiring modeled after `zdev/rl_design_colocate.sh`
