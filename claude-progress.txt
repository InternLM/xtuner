## Key Design Decisions (for coding agent to know)

### Pack Config Format Change
OLD (current code — WRONG, do not use):
  [dataset_id(int), sample_idx, token_start, token_end]  — 4 elements

NEW (correct):
  JSONL: [dataset_path(str), sample_idx, char_start, char_end, token_start_offset]  — 5 elements
  Parquet: [path_id(int), sample_idx, char_start, char_end, token_start_offset] + "paths" list

### __getitem__ Change
OLD (current code — WRONG):
  Slices input_ids[t_start:t_end] manually

NEW (correct):
  Just calls dataset[sample_idx] and does consistency check:
  - char_start==-1: verify item is plain DataItem (no 'char_start' key)
  - char_start!=-1: verify item is LongTextDataItem with matching char_start/char_end/token_start_offset

### Parquet Format (uses jsonl.py helpers)
save_mixed_dict_to_parquet({
    "boundaries": np.array([0,1,3,...]),  # CSR, shape (num_packs+1,)
    "samples": [[path_id, s_idx, c_start, c_end, tok_off], ...],  # list[list[int]]
    "paths": ["ds0.jsonl", "ds1.jsonl"],  # path_id -> dataset_path
}, "pack_config.parquet")

### No Skip Strategy
Only: error / padding (for short) and error / truncate (for long)
"skip" is NOT supported.

### _sample_num_tokens Bug in Old Code
Old code does: ds.num_tokens[np.array(ds.sampled)]  <- WRONG double indexing
New code should do: ds.num_tokens[i]  where i is the logical sample index 0..len(ds)-1
(JsonlDataset already reindexes num_tokens by sampled in its __init__)
